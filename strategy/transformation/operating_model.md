# Operating Model

## From Functions to Value Streams

The traditional operating model organizes people by function (sales, marketing, operations, IT). The target model organizes teams around **value streams** — end-to-end responsibility for delivering customer outcomes.

### What Changes

| Dimension | Legacy Model | Target Model |
|:----------|:-------------|:-------------|
| Organization | Functional silos | Cross-functional teams |
| Decision-making | Hierarchical approval | Autonomous within guardrails |
| Knowledge management | SharePoint, email, decks | Content-as-Code (this workspace) |
| Quality assurance | Manual review cycles | Automated validation + AI review |
| Reporting | Monthly PowerPoint updates | Real-time dashboards + auto-generated reports |

## AI-First Principles

1. **Agents as team members** — AI agents handle documentation, analysis, and reporting
2. **Content-as-Code** — Strategy managed with software discipline (git, validation, CI)
3. **Skills over scripts** — Reusable AI capabilities encoded as skills, not one-off prompts
4. **Automation by default** — Hooks validate every edit, catalogs regenerate automatically

## Agentic Team Structure

Each team has access to AI agents that handle:
- Document creation and maintenance (see `document-lifecycle` skill)
- Strategy review and scoring (see `strategy-review` skill)
- Presentation design (see `presentation-design` skill)
- Workspace integrity (see `workspace-health` skill)
